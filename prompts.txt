PROMPT OUTLINE: MULTIMODAL VIDEO CAPTIONING PROJECT
=====================================================

This document outlines the sequence of prompts that were given to Cursor to implement
this multimodal video captioning and summarization repository. The prompts are organized
by implementation phases, comparative analysis, and ablation studies.

================================================================================
PART 1: INITIAL PLANNING AND ARCHITECTURE
================================================================================

PROMPT 1: Complete Implementation Plan and Phases
--------------------------------------------------
"Create a complete implementation plan for a multimodal video captioning and 
summarization system. The system should:
- Process video files to extract audio and visual information
- Use Whisper for speech-to-text transcription
- Use CLIP for visual embeddings
- Implement multimodal fusion using transformer architectures with cross-attention
- Generate time-synchronized captions (SRT/VTT format)
- Generate abstractive summaries with key points
- Provide a web interface (React frontend + FastAPI backend)
- Support multiple output formats (SRT, VTT, TXT, DOCX, PDF)
- Include speaker diarization
- Implement comparative analysis of different fusion strategies
- Include ablation studies to identify critical components

Break down the implementation into clear phases with specific deliverables for each phase.
Define the folder structure, dependencies, and technical stack."

PHASE 2: Core Processing Modules
-----------------------------------

PROMPT 4: Video Preprocessing Module
"Implement video preprocessing modules:
- VideoProcessor class to extract frames from videos at configurable FPS
- AudioExtractor class to extract audio tracks from videos
- Audio chunking functionality for processing long videos
- Metadata extraction (duration, resolution, FPS, codec, etc.)
- Save extracted frames and audio to organized directory structure
- Include error handling for various video formats"

PROMPT 5: Audio Processing Module
"Implement audio processing modules:
- SpeechToText class using OpenAI Whisper for transcription
- Support for different Whisper model sizes (base, small, medium, large)
- Extract word-level timestamps from transcriptions
- SpeakerDiarization class using pyannote.audio
- Assign speaker labels to transcript segments
- Generate speaker statistics
- Save transcriptions and diarization results to JSON files"

PROMPT 6: Visual Processing Module
"Implement visual processing modules:
- FrameExtractor class to extract and preprocess video frames
- VisualEmbeddings class using CLIP for visual feature extraction
- Slide transition detection for presentation videos
- On-screen text extraction using OCR (optional)
- Temporal alignment of visual embeddings with audio timestamps
- Save visual embeddings and metadata to organized structure"

PROMPT 7: Integration of Phase 2 Modules
"Integrate all Phase 2 modules into a cohesive processing pipeline:
- Update ProcessingService to orchestrate video, audio, and visual processing
- Implement progress tracking for each processing stage
- Add error handling and recovery mechanisms
- Save all intermediate outputs to structured JSON files
- Update API endpoints to use the integrated pipeline"


PHASE 3: Multimodal Fusion
---------------------------

PROMPT 8: Cross-Attention Mechanism
"Implement bidirectional cross-attention mechanism for multimodal fusion:
- CrossAttention class for unidirectional attention (audio→visual or visual→audio)
- BidirectionalCrossAttention class for mutual attention (audio↔visual)
- Multi-head attention implementation
- Support for different fusion methods (concatenation, addition, gated)
- Layer normalization and dropout for regularization
- Proper tensor operations and dimension handling"

PROMPT 9: Multimodal Transformer Architecture
"Implement the complete multimodal transformer architecture:
- PositionalEncoding class for temporal information
- FeedForward network layers
- MultimodalTransformerLayer with cross-attention and self-attention
- MultimodalTransformer stack with configurable depth (6 layers default)
- Support for 12 attention heads and 768 hidden dimensions
- Proper residual connections and layer normalization
- Configurable dropout rates"

PROMPT 10: Fusion Pipeline and Service
"Implement the fusion pipeline and service:
- FusionPipeline class to orchestrate fusion process
- Model loading and saving functionality
- Embedding preparation and tensor conversion
- Temporal alignment utilities for audio-visual synchronization
- FusionService class for high-level integration
- Integration with ProcessingService
- Save fused embeddings and fusion metadata to disk
- Error handling and fallback mechanisms"

PROMPT 11: Audio Embedding Extraction
"Extend the SpeechToText module to extract audio embeddings:
- Extract embeddings from Whisper encoder (before decoder)
- Preserve temporal information in embeddings
- Save audio embeddings to disk alongside transcriptions
- Add configuration options for embedding extraction
- Integrate with fusion pipeline"

- Caption refinement (capitalization, punctuation, merging short captions)
- Support for multimodal embeddings (future-proofing for Phase 3)
- Save generated captions to JSON
- Error handling and fallback to simple text-based approach"

PROMPT 13: Summarizer with T5
"Implement abstractive summarization using T5 model:

- Generate abstractive summaries from full transcripts
- Handle long transcripts with chunking strategy
- Extract key points using hierarchical summarization
- Integrate visual context into summaries
- Enhanced grammar and formatting improvements
- Save summaries and key points to JSON"

PROMPT 14: Integration of Generation Pipeline
"Integrate caption generation and summarization into processing service:

- VideoPlayer component with caption overlay support

PROMPT 17: Output Formatting

PROMPT 18: Evaluation Metrics Implementation

- MetricsCalculator class to compute all metrics
- CaptionEvaluator class for batch evaluation
- Support for multiple reference captions"

- SimpleFusion models:

  * AdditionFusion: Element-wise addition of embeddings

- BaseCaptionModel interface for all models
- Model registry system for managing models
- Consistent interface for all models"
